# Phase 3: 架构重构 + 卡片重设计 PRD

> 版本: v1.1 (2026-05-29)
> 状态: 待确认
> 变更: v1.1 加入排序功能、统一喜欢操作、首页热门产品、推演修正

---

## 一、核心目标

将 opcquan 从"信息展示工具"升级为"有互动、有生命力的创业者社区"。

**三个关键转变:**
1. 产品从"名片"变成"有生命力的页面"(图片 + 进展 + 评论)
2. 卡片从"信息堆砌 + 多按钮"变成"视觉吸引 + 单一快捷操作"(参考小红书)
3. 用户后台从分裂的"个人中心 + 设置"变成统一的"我的"

## 二、数据模型变更

### 2.1 Product 模型修改

```prisma
model Project {
  // 废弃字段(数据迁移后可删列)
  // tagline  String  ← 废弃,内容合并到 description

  // 新增字段
  images    String[]  // 产品截图 URL,最多 5 张,第一张为封面

  // 新增关系
  progresses Progress[]
}
```

### 2.2 新增 Progress 模型

```prisma
model Progress {
  id        String   @id @default(cuid())
  content   String   // 进展内容(纯文本或 Markdown,≤2000字)
  milestone String?  // 里程碑标签 ID(可选,复用现有 MILESTONES 常量)
  projectId String
  project   Project  @relation(fields: [projectId], references: [id], onDelete: Cascade)
  authorId  String
  author    User     @relation(fields: [authorId], references: [id])
  createdAt DateTime @default(now())

  @@index([projectId, createdAt])
}
```

### 2.3 数据迁移

```sql
-- 1. tagline 合并到 description
UPDATE "Project"
SET description = CASE
  WHEN description IS NULL OR description = '' THEN tagline
  WHEN tagline IS NOT NULL AND tagline != '' AND tagline != description THEN tagline || E'\n\n' || description
  ELSE description
END
WHERE tagline IS NOT NULL AND tagline != '';

-- 2. 现有 PROGRESS 类型帖子迁移到 Progress 表
INSERT INTO "Progress" (id, content, milestone, "projectId", "authorId", "createdAt")
SELECT p.id, p.content, p.milestone, p."projectId", p."authorId", p."createdAt"
FROM "Post" p
WHERE p.type = 'PROGRESS' AND p."projectId" IS NOT NULL;
```

### 2.4 互动功能统一定义

| 功能 | 对象 | 数据表 | 卡片上 | 详情页 |
|------|------|--------|--------|--------|
| ❤️ 喜欢 | 产品 | Favorite (userId_projectId) | 可快捷操作 | 可操作 |
| 💬 评论 | 产品 | Comment (projectId) | 只展示数量 | 可操作 |
| 👤 关注 | 人 | Follow | 可快捷操作 | 可操作 |
| ❤️ 喜欢 | 帖子 | Favorite (userId_postId) | 可快捷操作 | 可操作 |
| 💬 评论 | 帖子 | Comment (postId) | 只展示数量 | 可操作 |

**关键决策:全站统一"❤️ 喜欢",不区分点赞和收藏。废弃 Like 表,全部用 Favorite。**

数据迁移:将 Like 表数据合并到 Favorite 表(userId + postId),去重后删除 Like 表。

## 三、页面架构

### 3.1 路由变更

| 路由 | 当前 | 改为 |
|------|------|------|
| `/profile` | 旧版个人中心(580行 Client Component) | redirect → `/profile/[自己的username]` |
| `/profile/[username]` | 公开主页(新版) | 保持,纯展示,isOwnProfile 时显示"编辑"入口跳 `/settings` |
| `/settings` | 三模块(基本信息/卡片/产品) | 重构为"我的"后台,六个分类 |
| `/projects/[slug]` | 三 tab(介绍/进展/评论) | 左右分栏,进展改用 Progress 模型 |
| `/connect` | 直通车表单 | 字段调整(tagline→description,加图片上传) |

### 3.2 "我的"后台(`/settings`)左边栏

| 分类 | 路由 hash | 内容 |
|------|-----------|------|
| 我的主页 | `#profile` | 个人资料编辑 + tab(动态/喜欢/关注/粉丝) |
| 我的产品 | `#products` | 产品列表、添加/编辑/删除产品(含图片上传、进展管理) |
| 账号与安全 | `#account` | 邮箱验证、密码修改、邮件通知开关 |

**"我的主页"内部结构:**
- 顶部:个人资料编辑区(头像/名字/简介/城市/赛道/阶段,折叠"更多信息"含 skills/canOffer/lookingFor)
- 下方 tab 切换:我的动态 | 我的喜欢(产品/帖子 sub-tab)| 我的关注 | 我的粉丝
- 顶部有"预览公开主页 →"链接

### 3.2.1 "我的产品"列表交互

每个产品卡片展示:产品名 + 阶段标签 + 描述摘要 + 喜欢数/评论数

操作按钮:
- "编辑" → 展开内联编辑表单(名称/描述/阶段/网站/图片)
- "查看" → 新窗口打开公开详情页 `/projects/[slug]`
- "记录进展" → 弹出 Dialog(内容 + 可选里程碑)
- "删除" → 二次确认后删除

### 3.3 导航菜单调整

用户头像下拉菜单:
- "我的" → 跳 `/settings`(后台管理)
- "私信" → 保持
- "退出" → 保持

**删除:** "个人中心"菜单项(避免用户困惑两个入口的区别)

**"我的"后台顶部:** 加一个"预览我的公开主页 →"链接,跳转到 `/profile/[username]`。

## 四、卡片设计规范

### 4.1 人物卡片(广场创业者 tab)

```
┌─────────────────────────────────────┐
│ [头像]  名字  ✓认证                 │  ← 点击跳主页
│         赛道标签 · 城市             │
│         简介(一行截断)             │
│                                     │
│ 📦 产品名                           │  ← 点击跳产品详情页
│ 粉丝 12 · 产品 2                   │
│                        [+ 关注]     │  ← 唯一快捷操作
└─────────────────────────────────────┘
```

**交互规则:**
- 卡片整体(除关注按钮和产品名外)点击 → 跳 `/profile/[username]`
- 产品名:点击跳 `/projects/[slug]`(仅展示第一个产品)
- 关注按钮:登录后可操作,未登录跳登录页
- 保留赛道和城市标签
- **删除:** "查看主页"按钮、"联系TA"按钮

**布局:** 3-4 列等高网格,根据视觉效果调整。

### 4.2 产品卡片(广场 products tab)

```
┌─────────────────────────────────────┐
│ [封面图 / 渐变色块+首字]            │  ← 占卡片上部 ~40% 高度
├─────────────────────────────────────┤
│ 产品名                    阶段标签  │
│ 描述摘要(两行截断,80字)          │
│                                     │
│ [小头像] 创建者名字                 │
│                                     │
│ ❤️ 23   💬 5                        │  ← ❤️ 可点击
└─────────────────────────────────────┘
```

**交互规则:**
- 卡片整体(除 ❤️ 按钮外)点击 → 跳 `/projects/[slug]`
- ❤️ 按钮:快捷喜欢/取消喜欢(乐观更新 + API 调用)
- 创建者名字:点击跳 `/profile/[username]`
- **删除:** "查看详情"按钮、"联系创始人"按钮、外链图标

**双模式卡片(根据 images 数组是否为空自动切换):**

有图版:
```
┌─────────────────────────────────────┐
│ [封面图]                              │  ← 占卡片上部 ~40%
├─────────────────────────────────────┤
│ 产品名                    阶段标签  │
│ 描述摘要(两行截断,80字)          │
│ [小头像] 创建者名字                 │
│ ❤️ 23   💬 5                        │
└─────────────────────────────────────┘
```

无图版(更紧凑,不强制封面区域):
```
┌─────────────────────────────────────┐
│ 产品名                    阶段标签  │
│ 描述摘要(三行截断,120字)         │
│                                     │
│ [小头像] 创建者名字                 │
│ ❤️ 23   💬 5                        │
└─────────────────────────────────────┘
```

混排时有图/无图卡片交替出现,视觉不单调。

### 4.3 帖子卡片(广场动态 tab)

## 七、广场改造

### 7.1 默认 Tab 改为产品

Tab 顺序:**产品** | 动态 | 创业者

### 7.1.1 排序功能(新增)

三个 tab 右上角统一位置加排序下拉:

**产品排序选项:**
- 最新发布(默认)
- 最多喜欢
- 最近更新(有新进展的排前面)

**创业者排序选项:**
- 最新加入(默认)
- 最多粉丝
- 最近活跃

**动态排序选项:**
- 最新发布(默认)
- 最多互动(喜欢+评论)

样式:小字下拉"排序:最新发布 ▾",三个 tab 统一位置。

### 7.1.2 筛选区(按数据属性设计)

| Tab | 筛选项 |
|-----|--------|
| 产品 | 阶段(想法/开发中/已上线/有收入/已盈利)+ 方向(赛道) |
| 动态 | 类型(聊聊/求助/分享/找人)+ 话题标签 |
| 创业者 | 方向(赛道)+ 城市 + 阶段 |

### 7.2 通知横条

广场顶部加一个滚动通知条:
- 内容:最近 24h 内的事件("xxx 发布了新产品"、"xxx 记录了新进展"、"xxx 加入了 OPC圈")
- 样式:单行,左右滚动或淡入淡出切换
- 数据源:从 Notification 表或 Activity 表查最近事件
- **始终展示:** 无事件时显示默认文案"欢迎来到 OPC 创业者广场",避免布局跳动

### 7.3 首页“热门产品”横版滚动

- 当前：竖排列表“最新动态”
- 改为：横版滚动卡片展示“热门产品”
- 展示逻辑：按喜欢数排序取 top 10 产品
- 卡片内容：封面图（或渐变色块）+ 产品名 + 描述摘要 + ❤️ 数
- 移动端可左右滑动
- 区域标题改为“热门产品”，右侧“查看全部 →” 跳广圼产品 tab

### 7.4 首页“最新入驻创业者”缩小

- 保留该区块，但缩小为一行横排（头像 + 名字）
- 主视觉让给热门产品
- 让新注册用户仍有首页曝光

### 7.5 首页数据统计调整

- “1,000+ 创业者” → “5,000+ 创业者”

### 7.6 邮件通知规则优化

**原规则：** 每次被关注/被评论都发邮件（对用户骚扰过多）

**新规则：每日摘要（Daily Digest）**
- 每天最多发一封汇总邮件，包含当天所有互动（新关注/新评论/新喜欢）
- 发送时间：每天 20:00（通过 cron 触发 `/api/cron/daily-digest`）
- 当天无互动则不发
- 用户可在“账号与安全”中关闭邮件通知
- **废弃：** 即时发送的 `sendFollowEmail` 和 `sendCommentEmail`，改为只写入通知记录，由 daily-digest 统一发送
