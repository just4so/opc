## P0 — 功能 Bug（必修）

### R2-01: 社区列表多关键词搜索不生效
**目的：** 社区列表页客户端过滤和搜索页 API 行为一致
**现状：** 搜索页 API 已支持多关键词 AND，但社区列表页的客户端过滤还是单字符串匹配
**修复：** `communities-page-client.tsx` 过滤逻辑拆词，每个词 AND 匹配 name/city

### R2-02: 未登录时政策不可见
**目的：** 政策是公开信息，应该吸引未登录用户，不该藏起来
**现状：** 政策区块在 Layer 2 内，未登录时整块隐藏
**修复：** 把政策侧边栏从 `!isLoggedIn` 条件块中移出，始终渲染

### R2-03: 无联系方式的社区仍显示"该社区暂无联系方式"
**目的：** 无意义的空状态文案让页面显得粗糙
**现状：** `else` 分支渲染了"该社区暂无联系方式"
**修复：** 无联系方式时不渲染联系信息区块，只保留直通车入口

### R2-04: BP 上传 Failed to fetch + 大小限制
**目的：** 用户能正常上传 BP，20MB 覆盖大部分 PDF/PPT
**现状：** MAX_SIZE_MB = 10，且可能有 CORS/超时问题
**修复：** ① MAX_SIZE_MB 改为 20 ② 排查 R2 presigned URL 的 CORS 配置和超时

### R2-05: 广场引导条逻辑未生效
**目的：** 根据用户状态显示不同引导，推动转化
**现状：** `showInPlaza` 从 session 对象取，但 JWT token 里没这个字段，永远是 undefined
**修复：** 改用 `/api/user/card` 接口查询用户状态，或者把 showInPlaza 加到 JWT callback

### R2-06: 广场引导条"让 5 位创业者看到你"数字太小
**目的：** 社会证明要有说服力，数字太小反而负面
**修复：** 去掉具体数字，改为"让更多创业者看到你"

## P1 — 体验优化（建议修）

### R2-07: 首页底部"探索更多"和导航重复
**目的：** 首页底部应该引导高价值行动，不重复导航
**修复：** 改为两块：左边"立即行动"（找社区入驻/展示我的产品），右边"社区运营方联系"（保留邮箱）

### R2-08: 注册/登录页 "OPC" 文字换 logo
**目的：** 品牌一致性
**修复：** 替换为 `/logo.svg` 或 `/logo.png`（需确认 logo 文件路径）

### R2-09: 右上角"创建卡片"改为"注册"
**目的：** 新用户不知道"卡片"是什么
**修复：** `user-nav.tsx` 未登录时文案改为"注册"

### R2-10: 推荐社区单独一组放最前
**目的：** 重点社区优先展示，引导用户
**修复：** `communities-page-client.tsx` 分组逻辑中，先渲染 featured=true 的社区为"推荐社区"组

### R2-11: 后台二维码支持多个
**目的：** 社群二维码和直通车联系二维码是不同场景
**修复：** admin/settings 改为支持多个 key：`community_qrcode_url`（社群）、`connect_qrcode_url`（直通车）。直通车成功页读 connect_qrcode_url

### R2-12: 广场产品信息显示不全 + 筛选按视图区分
**目的：** 用户能看到完整产品信息，按需筛选
**修复：** ① 人视图卡片增加产品完整信息（tagline+阶段+网站链接）② 产品视图增加类型筛选（项目/需求/合作）③ 产品卡片增加更多字段展示
