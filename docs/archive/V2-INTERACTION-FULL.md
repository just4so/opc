# V2 全站交互优化方案（逐页深度分析）

> 每个页面先分析用户目标和视线路径，再决定加什么交互。
> 原则：**交互服务于用户目标，不是装饰。** 用户要找东西，就让找的过程更顺畅；用户要做决定，就让关键信息更突出。

## 2. 社区列表 `/communities`

### 用户目标
找到感兴趣的社区。用户会：扫一眼推荐 → 搜索/筛选 → 点进详情。

### 当前问题
- 省份折叠/展开是硬切（display:none → block），没有过渡
- 页面加载后所有社区卡片同时出现，缺少层次
- 列表/地图视图切换是硬切
- 搜索框 focus 状态不够明显

### 优化方案
| 交互 | 目的 | 实现方式 |
|------|------|---------|
| **省份展开高度过渡** | 展开不跳跃，体验更丝滑 | CSS `max-height` + `overflow: hidden` + transition 300ms，或用 `grid-template-rows: 0fr → 1fr` |
| **社区卡片进入视口时渐入** | 长列表不死板 | 给卡片容器加 ScrollReveal，stagger 模式让同一省份的卡片依次出现 |
| **搜索框 focus 微扩展** | 视觉聚焦到搜索 | focus 时宽度从 100% 微增到 102%（或加更明显的 ring），transition 200ms |
| **推荐社区卡片微光边框** | 和普通社区区分 | 推荐社区卡片 border 用 `primary/30`，hover 时 `primary/50` |

### 不做
- ❌ 列表/地图切换动画（Map 组件重绘成本高，加动画反而卡）
- ❌ 城市筛选 pill 动画（数量多，加了反而花）

## 3. 社区详情 `/communities/[slug]`

### 用户目标
判断这个社区值不值得入驻。视线路径：封面图 → 名称/地址 → 入驻友好度 → 福利/政策 → 联系方式 → 直通车按钮。

### 当前问题
- 页面信息密度很高（左侧内容+右侧卡片），全部同时加载，看着压迫
- 直通车浮动按钮在手机端始终可见，但没有"吸引注意力"的手段
- 侧边栏政策/地图/联系信息卡片之间没有视觉节奏

### 优化方案
| 交互 | 目的 | 实现方式 |
|------|------|---------|
| **左侧内容区块依次渐入** | 减轻信息压迫感，引导视线从上到下 | 每个 Card 包 ScrollReveal，delay 递增 |
| **右侧侧边栏整体渐入** | 侧边栏在主内容之后 200ms 出现 | 侧边栏 div 包 ScrollReveal delay=200 |
| **手机端直通车按钮进入动画** | 不是一直在那里，是滚过首屏后从底部滑入 | `position: fixed` + 监听滚动，超过 500px 时 `translateY(0)` 从 `translateY(100%)` 过渡 |
| **认领/收录 Dialog 打开动画** | 弹窗不硬切 | shadcn Dialog 已有 fade+scale 默认动画，确认是否生效即可 |

### 不做
- ❌ 封面图视差滚动（手机端体验差）
- ❌ 星级评分动态填充（过度设计）

## 4. 直通车 `/connect` 和 `/connect/[slug]`

### 用户目标
填完表单提交意向。这是**转化核心页**，每一步都要让用户觉得"很快就填完了"。

### 当前问题
- 步骤 1→步骤 2→成功页是硬切（setStep 直接换内容）
- 成功页没有任何"恭喜"的反馈，就像普通页面
- BP 上传过程没有进度指示

### 优化方案
| 交互 | 目的 | 实现方式 |
|------|------|---------|
| **步骤切换滑动过渡** | "翻页"感，暗示进度 | 步骤 1→2：当前内容 fadeOut + slideLeft（200ms），新内容 fadeIn + slideRight（200ms）。用 CSS animation + key 切换触发。 |
| **成功页 check 动画** | 提交后的正反馈 | 成功页顶部的 ✓ 图标做 scale(0→1.2→1) + opacity 弹性动画（400ms）。纯 CSS keyframes。 |
| **成功页留存钩子渐入** | 引导用户继续探索 | "去广场看看"/"完善卡片" 链接在成功动画结束后 600ms 渐入 |
| **BP 上传进度条** | 减少等待焦虑 | 上传时显示一个橙色进度条（CSS width transition），完成时变绿 |

### 不做
- ❌ Confetti 动画（太 playful，不符合 B2B 社区调性）
- ❌ 步骤条组件（只有两步，不值得加）

## 5. 创业者广场 `/plaza`

### 用户目标
浏览创业者/产品 → 找到感兴趣的人 → 联系。或者展示自己。

### 当前问题
- Tab 切换（人/产品/动态）内容是硬切
- 卡片网格全部同时出现
- 引导条（"创建你的名片"）没有吸引力
- 筛选器选中/清除没有反馈

### 优化方案
| 交互 | 目的 | 实现方式 |
|------|------|---------|
| **Tab 切换内容淡入** | 切换不突兀 | 切换 Tab 时内容区 opacity 0→1 + translateY(8→0)，200ms。用 key={mainTab} 触发 CSS animation |
| **卡片网格错落入场** | 浏览感更好 | ScrollReveal stagger 包裹卡片 grid |
| **引导条入口脉冲** | 吸引没有卡片的用户创建 | 引导条左侧的箭头图标做缓慢脉冲动画（scale 1→1.15→1，2s 循环） |
| **筛选器选中态过渡** | 选中/取消有反馈 | 筛选 pill 选中时 bg-color + scale 做 150ms transition（已有 transition-colors，加 scale(1.02)） |
| **空状态友好动画** | 搜索无结果时不冷冰冰 | 空状态图标做轻微浮动动画（translateY 0→-4→0，3s 循环） |

### 不做
- ❌ 无限滚动加载动画（当前是分页，不需要）
- ❌ 卡片翻转展示更多信息（复杂度高，手机端难做）

## 6. 用户主页 `/profile/[username]`

### 用户目标
了解这个人是谁、做什么的、值不值得联系。

### 当前问题
- 头部信息 + 产品 + 动态同时出现，没有引导
- 完善度进度条是静态的，没有"填充"感
- "联系TA"按钮没有足够的视觉权重

### 优化方案
| 交互 | 目的 | 实现方式 |
|------|------|---------|
| **头部→产品→动态依次入场** | 引导视线从"人"到"作品"到"活动" | 三个区块各包 ScrollReveal，delay 0/100/200 |
| **完善度进度条动画填充** | 进度可视化更生动 | 进度条 width 从 0% 到实际值，transition 800ms ease-out，进入视口时触发 |
| **"联系TA"按钮微光** | 吸引点击 | 按钮 hover 时加 `box-shadow: 0 0 20px rgba(249,115,22,0.2)` |
| **最近活跃时间脉冲点** | 表示在线/活跃状态 | 如果 lastActiveAt < 24h，名字旁加一个绿色小圆点做脉冲动画 |

### 不做
- ❌ 头像上传动画（不在这个页面编辑）
- ❌ 产品卡片翻转/展开（信息量已经够用）

## 7. 个人设置 `/settings`

### 用户目标
编辑自己的信息 → 保存。

### 当前问题
- 左侧导航切换右侧内容是硬切
- 保存成功/失败反馈是一行文字（message state），不够显眼
- 产品增删改没有动画

### 优化方案
| 交互 | 目的 | 实现方式 |
|------|------|---------|
| **区块切换淡入** | 不突兀 | 切换 section 时右侧内容 fadeIn 200ms |
| **保存成功 toast** | 明确的成功反馈 | 保存后顶部出现绿色 toast，3s 后自动消失（slideIn + fadeOut） |
| **产品新增/删除动画** | 操作有反馈 | 新增时卡片 fadeIn + scaleUp；删除时 fadeOut + scaleDown（200ms） |

### 不做
- ❌ 表单字段逐个入场（设置页用户是来编辑的，不需要"发现"信息）
- ❌ 实时保存（当前是手动保存，改实时涉及架构）

### 用户目标
浏览资讯标题 → 找感兴趣的 → 点进阅读。

### 当前问题
- 资讯列表和政策 Tab 切换硬切
- 文章详情页底部 CTA 不够醒目

### 优化方案
| 交互 | 目的 | 实现方式 |
|------|------|---------|
| **资讯卡片渐入** | 列表不死板 | ScrollReveal 包裹资讯列表 |
| **文章详情底部 CTA 入场** | 读完文章后引导行动 | CTA 区块用 ScrollReveal 从底部渐入，delay 200ms |

### 不做
- ❌ 文章内容打字机效果（这不是 AI 对话）
- ❌ Tab 切换动画（内容量大，加了反而慢）

## 13. 后台 `/admin/*`

### 分析
后台是给你（管理员）用的效率工具，不是给用户看的展示页。**后台不加入场动画。**

### 仅做
| 交互 | 目的 | 实现方式 |
|------|------|---------|
| **保存/操作成功 toast** | 操作反馈 | 统一 toast 组件（如果设置页做了，后台复用） |
| **表格行 hover** | 当前行高亮 | `hover:bg-surface-soft` transition 100ms |

### 不做
- ❌ Dashboard 数字计数动画（你看后台要的是即时信息，不是动效）
- ❌ 侧边栏展开动画（效率优先）

## 全局复用组件需求

从以上分析提炼出需要新建/增强的组件：

| 组件 | 用途 | 涉及页面 |
|------|------|---------|
| **Toast** | 保存/操作成功/失败反馈 | 设置、后台、直通车 |
| **AnimatedProgress** | 进度条动画填充 | Profile 完善度 |
| **TabTransition** | Tab 切换时内容淡入 | 广场、资讯 |
| **ExpandTransition** | 高度过渡展开/收起 | 社区列表省份分组 |
| ScrollReveal stagger（已有）| 卡片错落入场 | 社区列表、广场、雷达、资讯 |
| btn-press（已有）| 按钮按压 | 全站 |
| card-interactive（已有）| 卡片 hover | 全站 |
