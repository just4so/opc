## P0 - 上线前必修（Bug + 体验硬伤）

### 1. 项目网站链接 bug（Settings + Profile）
**问题：** 用户填 `www.baidu.com`，点击跳转到 `localhost:3000/www.baidu.com`
**修复：** `href` 加 `https://` 前缀补全逻辑
**文件：** `settings/page.tsx:774`, `profile/[username]/page.tsx`

### 2. 通知跳转逻辑错误
**问题：** 入驻意向状态变更通知，点击跳转到 `/settings` 而不是意向详情
**修复：** `INQUIRY_STATUS` 应跳转到 `/profile`（或意向详情页，待定）；`CARD_VIEWED` 应跳转到查看者主页（用 relatedId）
**文件：** `components/notifications/notification-panel.tsx:27` TYPE_NAV 映射

### 3. 联系方式模糊化缺乏引导文案
**问题：** 用户看到模糊电话，不知道为什么模糊、怎么解锁
**修复：** 模糊区域加文案 "提交入驻意向后可查看完整联系方式" + 直通车按钮
**文件：** `components/connect/contact-unlock.tsx`

### 4. 社区详情页直通车入口不统一
**问题：** 有些社区右侧没有直通车入口
**修复：** 所有社区详情页右侧统一放置直通车 CTA
**文件：** `app/(main)/communities/[slug]/page.tsx`

### 5. 直通车成功页微信号需更新
**问题：** 添加微信号信息不正确
**确认：** 阿良哥提供正确的微信号 → 我替换

### 6. 后台意向管理不能编辑和导出
**问题：** 状态只能一次性点击，不能改回；没有 CSV 导出
**修复：** 加 status dropdown + CSV 导出 API
**文件：** `app/admin/inquiries/`, `app/api/admin/export/route.ts`

## P1 - 上线前建议修（体验优化）

### 7. 首页"展示我的项目"CTA 调整
**问题：** 新访客不理解这个按钮的意义
**建议：** 改为 "我是创业者" 或 "加入创业者广场"，引导到注册或广场

### 8. 首页价值卡片"认证创业者"→ 点击进入直通车
**问题：** 未登录用户点了直接跳登录→直通车，路径不自然
**建议：** 未登录时这个卡片跳 `/plaza` 或跳注册页，而不是直通车

### 9. 导航名称优化
**当前：** 找社区 / 创业者广场 / OPC雷达
**待定：** 阿良哥想调整哪几个？（需要你的意见）

### 10. 首页社区运营方联系信息
**当前：** 微信：opcquan01
**修复：** 改为 cooperation@opcquan.com

### 11. 直通车社区选择带城市标签
**问题：** combobox 只显示社区名，不显示城市
**修复：** 下拉选项改为 "社区名 · 城市"
**文件：** `components/connect/connect-form.tsx`

### 12. 公众号信息不隐藏
**问题：** 社区公众号信息目前和电话一起模糊化了
**修复：** 公众号名称公开展示，只模糊电话号码
**文件：** `app/(main)/communities/[slug]/page.tsx` 联系方式区块

### 13. 广场缺少"编辑我的卡片"入口
**问题：** 用户在广场看不到编辑自己卡片的入口
**修复：** 已登录且 showInPlaza=true 的用户，卡片区顶部加 "编辑我的卡片 →" 链接到 /settings#card

### 14. Settings 页面布局优化
**问题：** 基础信息、创业者卡片、项目产品混在一起
**建议：** 拆成侧边栏二级导航：基本信息 / 创业者卡片 / 项目管理

### 15. 社区认领管理功能补全
**问题：** 后台 community claims 列表有了，但操作逻辑不清晰
**修复：** 加 status 字段（待处理/已联系/已完成）+ 处理按钮

### 16. 后台功能聚合
**问题：** 后台菜单项多且分散
**建议：** 待讨论具体合并方案

## P2 - 上线后迭代

### 17. 冷启动种子数据
**问题：** 广场只有 5 张卡片
**行动：** 批量插入 20-30 张种子卡片（覆盖不同城市/方向/阶段）

### 18. 注册→直通车字段重复
**问题：** 注册填了手机号/方向/阶段，直通车又填类似信息
**建议：** 直通车自动填充做得更彻底，减少用户感知重复

### 19. 直通车成功页留存钩子
**问题：** 用户拿到联系方式后没理由回来
**建议：** 成功页加 "去广场看看其他创业者" / "完善你的创业者卡片"

### 20. 项目/产品展示系统设计
**问题：** 项目信息只在 profile 页展示，没有独立的浏览入口
**建议：** 后续考虑项目 showcase 页面

### 21. 整体设计视觉统一
**问题：** 风格不统一，不够高级
**行动：** 基于 design.md token 统一所有页面视觉（计划在功能冻结后集中做）
