## 一致性检查发现的问题

### 1. 直通车表单重设计 → Inquiry 模型需扩展
**现状：** Inquiry 有 `introduction`/`stage`，没有产品名/tagline/产品阶段/产品网站
**方案：** 不改 Inquiry 模型。直通车提交时：产品信息写入 Project 表，bio/城市写入 User 表。Inquiry 只存留资核心信息（姓名/联系方式/意向社区）
**影响：** 直通车 API（POST /api/inquiries）需要重写提交逻辑

### 2. 通知→私信联动 → Message 模型需支持系统消息
**现状：** Message.senderId 是必填的 userId，系统消息没有 sender
**方案：** 新增一个系统用户（username: "system"），系统消息用这个用户发送
**影响：** 需要 seed 一个系统用户 + 修改通知触发逻辑

### 3. 广场三视图 → 产品视图需要新 API
**现状：** 广场只查 User，没有独立查 Project 的 API
**方案：** 新增 GET /api/projects（公开，分页，带筛选）
**影响：** 新增 API + 广场前端重写

### 4. 首页价值卡片第二张跳转
**现状：** 跳 /connect
**讨论确认后方案：** 未登录跳 /register，已登录跳 /settings#card（和右CTA一致）

### 5. 后台系统设置存图片
**现状：** /admin/settings 可能没有图片上传功能
**方案：** 用 R2 存储二维码图片，admin settings 加上传 UI + API

### 6. lastActiveAt 更新机制
**讨论确认了卡片/profile 要显示"最近活跃"，但 lastActiveAt 目前没有自动更新逻辑**
**方案：** 在 auth callback 的 jwt 阶段或 middleware 里更新 lastActiveAt

## 验收标准（每个 Phase 完成后）

1. **`npm run build` 零错误**
2. **git diff 确认只改了该改的文件**
3. **curl / 浏览器逐项验证验收标准**
4. **对照 V2-DISCUSSION.md 确认实现与讨论结论一致**
5. **commit 格式：`feat(v2.1): Phase X — 描述`**
6. **更新 CLAUDE.md 当前进度**
7. **飞书通知包含"已对照讨论结论验收"**
